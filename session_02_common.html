Part 01 · 00:00 – 05:00

Stitch, 이렇게 시작해요 🚀

처음 써보는 분들을 위한 빠른 시작 가이드. 3분이면 첫 번째 시안이 나옵니다.

시작 전 준비
1
stitch.withgoogle.com 접속
구글 계정으로 로그인. 별도 설치 없이 웹 브라우저에서 바로 사용 가능
2
새 프로젝트 만들기
홈 화면에서 New Project 클릭. 모바일·웹 중 목적에 맞게 선택
3
AI 모델 선택
Gemini 2.5 Flash — 빠른 생성 (월 350회), Gemini 2.5 Pro — 고품질 생성 (월 50회). 실습은 Flash로 시작 추천!
4
프롬프트 입력 후 Generate
원하는 화면을 설명하고 Generate 클릭. 약 60~90초면 시안이 나와요
좋은 프롬프트의 3요소
😞 이렇게 하면
"앱 화면 만들어줘"

→ 맥락 없이 일반적인 화면만 생성
✅ 이렇게 하면
① 목적 + ② 화면 구성 + ③ 분위기·색상

"M몰 앱의 추석 기획전 메인 화면. 상단 배너, 카테고리 탭, 상품 카드 그리드 구성. 프리미엄한 느낌, 다크 네이비 컬러"
Part 02 · 05:00 – 18:00

라이브 실습 — 3단계로 완성 ⚡

진행자가 실시간으로 Stitch를 조작합니다. 어떻게 발전시키는지 과정을 주목해보세요!

Demo 14분
M몰 앱 기획전 메인 화면 생성 📱
텍스트 설명만으로 M몰 스타일의 기획전 메인 화면을 즉석에서 만들어봅니다.
Stitch에 입력하는 프롬프트
모바일 앱 화면 · M몰 기획전
현대카드 M몰 앱의 추석 기획전 메인 화면을 디자인해줘.

화면 구성:
- 상단: 추석 기획전 풀 배너 (이미지+텍스트)
- 중단: 카테고리 탭 (뷰티/식품/가전/패션)
- 하단: 상품 카드 그리드 (2열, 포인트 가격 표시)

분위기: 프리미엄하고 고급스러운 느낌
색상: 딥 네이비 배경, 골드 포인트 컬러
스타일: 현대카드 브랜드에 맞는 미니멀하고 세련된 디자인
Demo 25분
AI Chat으로 화면 수정하기 🔧
처음 생성된 시안을 채팅으로 수정하는 방법을 보여줍니다. 생성보다 수정이 더 중요해요!
후속 수정 프롬프트 예시
채팅창에 이어서 입력
수정 요청 1) 배너 영역을 더 크게 키우고, "최대 50% 포인트 적립" 문구를 뚜렷하게 넣어줘

수정 요청 2) 상품 카드에 "한정수량" 뱃지와 남은 수량 표시를 추가해줘

수정 요청 3) 전체적인 폰트를 더 굵고 읽기 쉽게 바꿔줘
💡 수정할 때 구체적인 위치와 내용을 함께 말해줄수록 원하는 결과가 나와요!
Demo 35분
다음 화면 추가 → 프로토타입 연결 🔗
상품 상세 화면을 추가하고 메인 화면과 연결해 클릭 가능한 프로토타입을 만들어봅니다.
1
상품 상세 화면 추가
왼쪽 패널에서 + Add screen → "M몰 상품 상세 화면, 포인트 가격·구매 버튼·상품 이미지 포함" 입력
2
화면 간 연결 (Stitch)
메인 화면의 상품 카드 컴포넌트 선택 → Link to screen → 상품 상세 화면 선택
3
Play로 프로토타입 미리보기
우측 상단 ▶ Play 클릭 → 실제로 클릭해서 화면 전환되는 프로토타입 확인!
Demo 43분
Ideate 모드 — 아이디어를 빠르게 여러 방향으로 펼치기 💡
Ideate 모드는 하나의 프롬프트로 다양한 디자인 방향의 시안을 동시에 여러 개 생성해줍니다. 기획 초기 단계에서 방향성을 빠르게 탐색할 때 가장 유용해요.
1
Ideate 모드 진입
프롬프트 입력창 하단에서 Ideate 탭 선택. 생성 전 여러 콘셉트를 동시에 받아보는 모드
2
큰 방향만 잡아서 입력
세부 사항 없이 목적과 분위기 위주로 입력하면 AI가 다양한 해석으로 변형안 제시. 예: "M몰 포인트 적립 프로모션 배너, 프리미엄 느낌"
3
마음에 드는 시안 선택 후 발전
생성된 여러 안 중 방향성이 맞는 것을 골라 Edit으로 진입 → 채팅으로 세부 수정 진행
💡 기획·MD 파트에서 팀 리뷰 전 방향성 후보를 뽑을 때 특히 유용해요. "이 중에 어떤 느낌이 좋아?" 논의 자료로 바로 활용 가능!
Demo 53분
Redesign 모드 — 기존 화면을 AI로 개선하기 🔁
Redesign 모드는 기존 화면 캡처 이미지를 업로드하면 AI가 이를 분석해 개선된 시안을 생성해줍니다. 현재 M몰 화면이나 경쟁사 레퍼런스를 기반으로 빠르게 개선안을 만들 수 있어요.
1
Redesign 모드 진입 & 이미지 업로드
프롬프트 입력창에서 Redesign 탭 선택 → 개선하고 싶은 화면 캡처 이미지를 드래그하거나 첨부
2
개선 방향 프롬프트 입력
현재 화면의 어떤 부분을 어떻게 바꾸고 싶은지 지시. 예: "전반적인 레이아웃은 유지하되, 버튼 CTA를 더 강조하고 가독성을 높여줘"
3
원본 vs 개선안 비교
생성 후 원본 이미지와 나란히 비교하며 어떤 부분이 바뀌었는지 확인. 필요하면 채팅으로 추가 수정
💡 경쟁사 앱 화면을 업로드해서 "M몰 스타일로 재해석해줘"도 가능해요. 벤치마킹 + 시안 초안을 한 번에!
결과물 공유하기
만든 시안을 팀에 공유하는 3가지 방법 📤
📋 Figma로 내보내기
Export → Copy to Figma. 디자인팀과 협업하거나 추가 편집이 필요할 때
💻 코드로 내보내기
Export → Download. HTML/CSS 코드 추출로 개발팀에 바로 전달 가능
🔗 링크로 공유
Share → Public link. 팀원들이 브라우저에서 바로 프로토타입 확인 가능

공유 & 마무리 🎤

만든 시안을 공유하고, 오늘의 핵심을 정리합니다.

공유 질문2분
1~2명이 화면 보여주세요 🙋
1) 어떤 화면을 만들어봤나요? 실제로 쓸 수 있는 수준이었나요?
2) 수정 요청을 몇 번 했나요? 어떻게 달라졌나요?
3) 기획·디자인 과정에서 어떻게 활용하면 좋을 것 같나요?
오늘 꼭 기억할 것1분
  • 목적 + 화면 구성 + 분위기·색상 — 이 3가지를 넣으면 원하는 시안이 나와요 🎯
  • 처음 결과가 별로면 채팅으로 구체적인 위치와 내용을 지정해서 수정하세요 🔧
  • 화면을 이어 붙이면 클릭 가능한 프로토타입이 돼요. 팀 공유·테스트에 바로 활용 가능 🔗
